# Aura 人物角色线设计草案 V1

> 状态：Draft for Production  
> 适用范围：Aura Chrome 插件角色皮肤线  
> 目标：在不破坏 Aura “低打扰陪看”主线的前提下，引入第一批人物角色皮肤，并建立可复用的人物角色生产模板。

## 1. 这条角色线的定位

Aura 的人物角色线不是用来替代默认宠物主线，而是作为更强题材感和更高审美吸引力的主题分支。

角色线的职责是：
- 提供比宠物更强的剧种氛围感
- 承担古装 / 古偶 / 权谋向题材的审美表达
- 形成“可收藏、可切换、可主题化”的高吸引力皮肤

角色线不应该变成：
- 写实影视角色复刻
- 具体明星脸
- 小尺寸下信息过载的海报缩略图
- 高打扰、抢戏、抢字幕的页面装饰

## 2. 统一设计原则

## 2.1 角色形态原则

- 首批人物角色统一采用 `Q版 / 半身 / 角落友好构图`
- 不做写实真人脸，不做高精五官特写
- 不做全身站姿主图
- 不做复杂背景，不做叙事场景图
- 右下角主体优先使用 `2.5头身到3头身` 的精致卡通比例

## 2.2 运行时资产原则

每套人物皮肤仍严格遵守 Aura 当前双槽模型：

1. `top-left badge`
2. `bottom-right companion`

不要把人物角色做成整张海报再硬裁成两块。左上和右下必须一开始就按不同职责设计。

## 2.3 动作原则

人物角色线必须和宠物线的动作语言区分开。

宠物线的代表动作：
- 呼吸
- 探头
- 耳朵动
- 爪子搭边

人物线的代表动作：
- 视线偏移
- 发丝 / 流苏 / 披风微动
- 扇子 / 玉佩 / 甲片的小动作
- 轻抬眸、轻侧身、轻收袖

换句话说：
- 宠物像“陪你一起趴着看”
- 人物像“陪你一起静静观剧”

## 2.4 风格边界

- 不做具体 IP 角色
- 不做过强二创指向
- 不做明显影视剧名场面复刻
- 不做血腥兵器冲脸
- 不做大面积黑色压暗视频边角

## 3. 首批角色定义

首批角色固定为两套 archetype：

1. 古代美女
2. 古代将军

这两套先作为 Aura 人物线的基础模板，不引入“谋士 / 少侠 / 侠女 / 皇子 / 魔尊”等更多细分类。

## 3.1 古代美女

### 人设一句话

一个温柔、克制、精致、有陪伴感的古风贵女角色，适合古偶、言情、权谋中的柔性观看氛围。

### 角色关键词

- 温柔
- 从容
- 清雅
- 月色感
- 珠钗
- 团扇 / 花枝 / 流苏
- 轻粉、暖金、玉白

### 适合承载的情绪

- 陪伴
- 安静看戏
- 轻惊讶
- 轻含笑
- 温柔等待

### 不适合的方向

- 妖艳浓妆
- 大红大紫婚嫁主妆
- 过强宫斗攻击感
- 写实美女脸
- 夸张卖萌表情

### 建议视觉元素

- 发饰：步摇、珠钗、玉簪
- 手持：团扇、花枝、小灯
- 服饰：水袖、披帛、层叠领口
- 光感：月华、柔金、微粒花影

### 推荐配色

- 主色：`#f3dbe7` / `#f6e9dd`
- 点缀：`#d8b36a`
- 冷高光：`#f8f7ff`
- 辅助色：`#d6b7c8`

## 3.2 古代将军

### 人设一句话

一个克制、沉稳、有保护感和观察感的古风将军角色，适合古装、权谋、家国、热血成长向题材。

### 角色关键词

- 沉稳
- 守护感
- 清冷克制
- 肩甲
- 披风
- 玉佩 / 剑穗 / 战旗纹
- 深青、暗红、古金

### 适合承载的情绪

- 警觉
- 守望
- 稳定陪伴
- 轻微审视
- 克制反应

### 不适合的方向

- 纯战斗怒吼状态
- 血迹、杀气、过猛兵器前刺
- 全黑铠甲导致小尺寸糊成一团
- 写实男明星脸
- 大幅度热血战斗 pose

### 建议视觉元素

- 发型：束发、高马尾、冠饰
- 手持：折扇、剑柄、令牌，不建议整把武器外露
- 服饰：肩甲、披风、内袍层次
- 光感：冷金边、高光甲片、微风披风边缘

### 推荐配色

- 主色：`#314055` / `#546070`
- 点缀：`#c89d58`
- 强调色：`#7a3033`
- 高光：`#d8dceb`

## 4. 左上和右下的构图定义

## 4.1 总原则

### 左上

- 角色线索 / 阵营感 / 题材徽记
- 绝不直接放完整人物头像
- 目标是“缩到很小仍能一眼看懂”

### 右下

- 半身角色挂件
- 主情绪和主道具只保留一项
- 目标是“有陪伴感、能动、轮廓清楚”

## 4.2 古代美女构图

### 左上应该怎么画

左上建议做成：
- `团扇 + 花枝 + 珠钗` 的轻徽章
- 或 `月轮 + 花瓣 + 流苏` 的角标组合
- 或 `半枚玉佩 + 金月纹` 的 Aura 化古风角标

左上不应该做成：
- 美女完整脸
- 缩小版立绘
- 复杂宫殿背景
- 大面积矩形牌匾

### 左上构图建议

- 主体集中在画布左上 `60% x 60%`
- 右下留白
- 单主图形 + 一到两个辅助装饰
- 优先做 “圆形 / 扇形 / 半月形” 的轮廓，不做大矩形

### 右下应该怎么画

右下建议做成：
- 半身古风美女从右下角安静探出
- 一只手轻持团扇或花枝
- 头部略微偏向屏幕中心
- 眼神温和而不是直勾勾盯用户

### 右下构图建议

- 主体占画布右下 `70% x 75%`
- 头部位于画布偏右中上区域
- 左上留足空白
- 不要把长裙、长袖铺满整张画布

### 右下最适合的两个版本

1. `团扇贵女`
   - 扇子遮一部分肩颈或胸前
   - 适合左上花瓣 / 月影与右下轻呼吸配合
2. `花枝贵女`
   - 一支花枝作为视觉锚点
   - 更温柔，但动作变化空间小于团扇版

首发优先推荐：`团扇贵女`

## 4.3 古代将军构图

### 左上应该怎么画

左上建议做成：
- `将军令牌 + 甲纹 + 披风角`
- 或 `战旗纹 + 玉佩结 + 古金徽记`
- 或 `半枚肩甲 + 风纹` 的阵营感角标

左上不应该做成：
- 将军头像
- 整把武器插在角上
- 大片深色旗帜压黑视频

### 左上构图建议

- 轮廓优先做 “盾形 / 旗角 / 令牌形”
- 单一主徽记 + 一处古金高光
- 远看像“军阵身份标”，近看才看出细节

### 右下应该怎么画

右下建议做成：
- 半身将军从右下角侧身露出
- 披风和肩甲形成清晰轮廓
- 一只手可扶剑柄或持令牌，但不要露出完整兵器
- 视线偏向视频中心，呈“守望”而不是“攻击”

### 右下构图建议

- 头肩轮廓优先于武器轮廓
- 披风只做一角，不要大面积铺开
- 甲片高光必须清楚，否则小尺寸会糊

### 右下最适合的两个版本

1. `扶剑少将`
   - 一只手靠近剑柄
   - 有力量感，识别度高
2. `持令将军`
   - 手持令牌或折扇
   - 更文气，更像权谋守望角色

首发优先推荐：`持令将军`

## 5. 动作语言如何和宠物区分

## 5.1 总差异

宠物动作重点是身体和探头感。

人物动作重点是：
- 眼神
- 发丝
- 配饰
- 衣袖 / 披风 / 扇子的小幅运动

### 宠物线不要照搬到人物线的动作

- 大幅上下弹跳
- 明显探爪
- 大头左右探出
- 过度卖萌表情切换

### 人物线优先动作

- 轻眨眼
- 轻转眸
- 轻抬扇 / 轻收袖
- 发丝 / 流苏 / 披风边缘轻晃
- 局部金属或珠饰扫光

## 5.2 分状态动作建议

| 状态 | 古代美女 | 古代将军 |
|---|---|---|
| `idle` | 左上花瓣 / 月影低频飘动，右下轻微呼吸 | 左上桃枝 / 冷雾低频飘动，右下轻微呼吸 |
| `paused` | 氛围粒子放缓，右下几乎静止 | 氛围粒子放缓，右下几乎静止 |
| `attention` | 右下轻微亮度和 scale 响应，不换帧 | 右下轻微亮度和 scale 响应，不换帧 |
| `controlsVisible` | 降低配饰摆动，保留极轻生命感 | 降低披风和扫光，动作收敛 |
| `adActive` | 降低亮度与高光，停掉显眼反馈 | 降低甲片高光与披风动势，停掉显眼反馈 |

## 5.3 首批动效建议

人物角色第一版不再使用右下替换帧。当前更稳定的路线是：

- 右下只保留一张高质量单图
- 右下只做低频呼吸、轻微位移和轻微 glow
- 左上承接主要氛围动效，例如花瓣、月影、雨丝、星点
- 回看点、暂停等轻反馈放到右下角色附近短暂出现

### 古代美女建议

- 右下：团扇贵女半身单图，轮廓轻薄、不要贴纸感
- 左上：月轮、扇纹、流苏或花瓣落下
- 动效：左上花瓣 / 流苏氛围为主，右下只保留轻微呼吸

### 古代将军建议

- 右下：文人将军半身单图，执杯或扶案，保持低打扰
- 左上：桃枝、月影、旗角或冷雾
- 动效：左上花瓣 / 冷雾为主，右下只保留轻微呼吸

## 6. AI 生图尺寸与文件命名

## 6.1 统一尺寸

### 左上角标

- 比例：`1:1`
- 首选尺寸：`1536 x 1536`
- 备选：`1024 x 1024`

### 右下主图

- 比例：`4:5`
- 首选尺寸：`1536 x 1920`
- 备选：`1536 x 2048`
- 低配备选：`1024 x 1280`

## 6.2 推荐文件命名

### 古代美女

- `skin-ancient-beauty-top-left.png`
- `skin-ancient-beauty-bottom-right.png`

### 古代将军

- `skin-ancient-general-top-left.png`
- `skin-ancient-general-bottom-right.png`

## 6.3 AI 生图要求

### 通用要求

- 透明背景
- 单主体
- 无完整场景背景
- 干净描边
- 暗场和亮场都能看清轮廓
- 同套皮肤左上和右下必须同一画风

### 左上角标 Prompt 约束

- “ornament badge”
- “corner-safe composition”
- “single decorative emblem”
- “transparent background”
- “no full portrait”
- “small-scale readable silhouette”

### 右下人物 Prompt 约束

- “bottom-right overlay companion”
- “half-body 2D anime character”
- “transparent background”
- “clean silhouette”
- “leave top-left negative space”
- “not full-body”
- “not poster”
- “not scene illustration”

## 6.4 推荐出图批次

每个角色建议按以下顺序出图：

1. 左上角标母图
2. 右下 base 主图
3. 可选预览图

## 7. 哪个角色更适合先做第一套真正的人物皮肤

### 结论

首发优先推荐：`古代美女`

### 原因 1：更符合 Aura 当前“低打扰陪伴”气质

古代美女的视觉语言天然更适合：
- 温柔
- 安静
- 陪伴
- 常开不烦

这和 Aura 现阶段的产品气质最一致。

### 原因 2：和现有古偶题材最自然衔接

Aura 当前已经有 `古偶 / 古装 / 花瓣 / 金粉` 的皮肤语义基础。

把第一套人物角色落成古代美女，会比古代将军更顺滑，不需要重新解释产品方向。

### 原因 3：动作更容易先做漂亮

古代美女更容易做出精致但克制的动作：
- 花瓣落下
- 流苏氛围
- 月影轻移
- 右下轻微呼吸

这些都很适合 Aura 的小幅低频动效系统。

### 原因 4：用户接受度更广

对于追剧、古偶和轻权谋用户来说，古代美女更容易成为：
- 愿意手动切换的主题皮肤
- 愿意长期常开的角色
- 更容易接受的“人物陪看”第一印象

## 8. 推荐执行顺序

### 第一套人物皮肤

`古代美女`

#### 首发形态

- 左上：月轮团扇角标
- 右下：团扇贵女半身 companion
- 动效：左上月影 / 花瓣，右下单图轻微呼吸

### 第二套人物皮肤

`古代将军`

#### 次发形态

- 左上：令牌甲纹角标
- 右下：持令将军半身 companion
- 动效：左上旗角 / 桃枝 / 冷雾，右下单图轻微呼吸

## 9. 第一轮落地检查清单

人物角色线正式开始生产前，必须先确认：

1. 仍遵守 Aura 当前双槽运行时结构
2. 右下只做半身，不做全身人物
3. 不使用具体影视人物脸
4. 左上不放完整人物头像
5. 不接入右下替换帧，避免双影、串帧和跳帧
6. 在 `Skin Studio` 中至少验证：
   - `windowed`
   - `paused`
   - `attention`
   - `controlsVisible`
   - `adActive`

## 10. 一句话执行建议

如果只做一套真正的人物首发皮肤，就先做：

> `古代美女 · 团扇贵女`

原因是它最符合 Aura 当前的产品气质、最容易做出优雅克制的小动作、也最适合作为人物角色线的第一印象。
