# Aura 腾讯视频手动回归清单

这份清单服务于当前 Aura 的腾讯视频主链。每次修改运行时、皮肤 contract、布局或动效后，至少过一遍这里的关键场景。

## 基础链路

1. 打开腾讯视频播放页，Aura 正常识别并显示左上角和右下角挂件
2. 打开 popup，状态、当前皮肤、观影模式与页面实际状态一致
3. 使用 `Command + Shift + A` 或 `Ctrl + Shift + A` 关闭 Aura，挂件立即消失
4. 再次开启 Aura，挂件恢复显示且不需要手动刷新页面
5. 在关闭状态下刷新页面，挂件不应残留或重现

## 播放场景

1. 普通窗口态下，左上角不压剧名核心区，右下角不压字幕主区域
2. 网页全屏下，挂件边距合理，没有贴边或过度缩放
3. 原生全屏下，挂件仍保持角落锚定，不侵入控制区
4. 控件浮现时，右下角挂件应自动上移并降低存在感
5. 暂停态时，动效频率明显放缓但不应卡住

## 异常与弱化场景

1. 广告态下，挂件整体弱化，不能比广告内容更抢眼
2. 非播放页进入详情页、选集页时，不应提前显示挂件
3. 从播放页跳到非腾讯页面时，不应报错或残留 DOM
4. 页面重新导航到另一集时，popup 状态应跟随新页面更新
5. 同时打开两个腾讯播放页来回切换时，popup 必须跟随当前活动页状态，不能串到另一个标签页的剧名、皮肤或运行态

## 皮肤联调

1. 在 `popup -> 打开皮肤预览台` 中验证素材比例和 `motionPreset`
2. 至少检查 `windowed / web-fullscreen / native-fullscreen`
3. 至少检查 `paused / controls-visible / ad-active`
4. 新皮肤上线前确认：
   - 左上素材不会形成厚重矩形黑块
   - 右下角色在亮场中仍能看清轮廓
   - `quiet / standard / lively` 三档下观感仍自然

## 通过标准

- 页面状态、popup 状态、storage 状态一致
- 关闭后无残留，刷新后不反复
- 新皮肤只改素材和 manifest，不改 runtime
