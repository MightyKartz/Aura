# Aura 收口重构方案（2026-04-11）

## 目标

本轮不是继续给现有实现打补丁，而是把 Aura 当前的腾讯视频 MVP 收成一条更清晰、更克制、也更容易扩展的主线。

目标只有三件事：

1. 让挂件从“能显示”提升到“像产品”
2. 让运行时从“勉强兼容”提升到“结构干净”
3. 让仓库从“保留历史包袱”提升到“当前主线清晰”

---

## 现状判断

结合真实截图与当前代码，问题主要集中在四层：

### 1. 视觉层

- 右下角色在亮场下存在感偏强，容易像贴纸浮在正片上
- 左上角标与右下角色的同套感还不够强
- 当前挂件尺寸、透明度、边距更多是工程试探值，还不是稳定模板
- 控件态、广告态、暂停态下的表现还不够克制

### 2. 运行时层

- 当前 `content.js` 逻辑过重，已经堆到一千多行
- 为适配 iframe/video 的兼容链仍保留了较多复杂逻辑
- popup 状态和内容脚本状态之间仍有历史兼容痕迹
- 当前代码仍偏“为了识别而识别”，不够聚焦“顶层播放器壳渲染”

### 3. Popup 层

- popup 当前更像调试台，不像普通用户控制面板
- 用户真正需要的是开关、模式、皮肤、强度和简要状态
- 调试信息应该隐藏，不应与主配置处于同等层级

### 4. 仓库层

- 旧主题资源仍在仓库中保留
- 旧映射和兼容别名仍散落在多个文件里
- 黑边时代的历史资源与当前角落挂件主线并存，造成理解负担

---

## 设计方向

### 核心方向

Aura 的 V1 应被定义为：

> 一个会陪你追剧、但不会抢戏的角落挂件插件。

### 视觉原则

1. 挂件应该“贴边存在”，不应“漂浮抢戏”
2. 右下主角色是陪伴主体，左上角标是题材提示，不应主次倒置
3. 三类皮肤共享统一模板，只在气质和配件上变化
4. 动效重点是呼吸感与生命感，不是持续表演
5. popup 的氛围感应和挂件一致，避免割裂

---

## 实施方案

## 一、重构内容脚本主链

### 改造方向

将运行时明确收敛为：

- 顶层页面优先
- 腾讯视频播放器壳优先
- 角落装饰优先

### 要做的事

1. 重写播放器容器检测链
   - 以顶层可见播放器壳和内容 overlay 为优先目标
   - 不再让“找到可见 `video`”成为唯一前提

2. 移除不再必要的上下文桥接复杂度
   - 删除为 iframe 状态同步保留的冗余逻辑
   - 删除只为旧识别链服务的本地状态中转

3. 收口状态模型
   - 保留 MVP 真正需要的状态字段
   - 删除历史方案中不再有产品价值的诊断兼容

4. 把视觉行为做成模板化布局
   - 统一角落安全边距
   - 统一挂件尺寸范围
   - 统一控件态/暂停态的衰减逻辑

### 预期结果

- `content.js` 职责更单一
- 运行时更偏“播放器壳装饰器”而不是“通用视频侦测器”
- popup 状态更稳定

---

## 二、收口挂件视觉模板

### 改造方向

将当前“图片 + 一些散落数值”的方式，收成统一模板系统。

### 要做的事

1. 建立统一的角落布局比例
   - 左上角标更小、更贴边
   - 右下角色更靠边、更像陪伴物

2. 增加状态化衰减
   - 控件出现时减弱
   - 暂停时更安静
   - 广告态尽量弱化

3. 统一动效节奏
   - 安静、标准、活泼三档仍保留
   - 但动画幅度统一收小，避免“常开疲劳”

4. 统一皮肤描述逻辑
   - 默认 / 悬疑 / 古偶 / 热血 的文本、标签和推荐逻辑统一口径

### 预期结果

- 三套皮肤看起来是同一产品，而不是三张不同插画
- 窗口态下更贴近“陪伴感”

---

## 三、重做 popup 信息架构

### 改造方向

popup 由“开发诊断台”收为“用户配置面板 + 可折叠调试层”。

### 要做的事

1. 精简首屏信息
   - 保留站点支持、当前剧集、当前皮肤、运行状态
   - 去掉视觉上冗余但无决策价值的行

2. 强化主操作区
   - 开关
   - 皮肤模式
   - 手动皮肤
   - 动效强度

3. 折叠调试层
   - 调试信息默认隐藏
   - 仅在用户主动展开时可见

4. 统一文案
   - 从“技术术语”转成“用户能理解的状态文案”

### 预期结果

- popup 更像正式产品
- 调试不妨碍使用

---

## 四、删除旧路线遗留

### 删除范围

1. 删除未使用的旧主题资源
2. 删除旧 `builtin-themes` 相关兼容心智在当前主线中的残留影响
3. 删除不再服务当前 MVP 的旧映射键值
4. 删除为历史黑边方案留下的命名和注释干扰

### 原则

- 仍用于文档存档的内容可以保留在文档中
- 不再用于构建和运行时的资源、分支和映射应删除

---

## 实施顺序

1. 写入设计上下文并固定方案
2. 重构 `content.js`
3. 收口 `content.css`
4. 重做 `popup.html / popup.css / popup.js`
5. 清理 `background.js / manifest.json` 中不再需要的复杂项
6. 删除未使用资源
7. `npm run build` 验证

---

## 本轮验收标准

### 运行时

- 腾讯视频窗口态稳定显示
- popup 状态能正确反映是否已显示
- 无需依赖 iframe 子页面状态回传

### 视觉

- 右下角色不明显压正片主体
- 左上角标不挤占剧名主信息
- 控件出现时挂件存在感明显下降

### 代码

- 主链明显比当前更短、更单一
- 未使用旧资源从当前主线中移除
- 不再新增“临时兼容状态”
