# Aura 盈利模式策略 V1

## 核心判断

Aura 最适合的盈利路线不是广告，也不是一开始做订阅，而是：

> 免费插件获取用户，靠高质量皮肤包、定制工具、后续创作者市场和官方联名赚钱。

Aura 的核心价值是“好看、陪伴、不打扰”。如果过早加入广告、强订阅或高成本 AI 对话，会破坏产品气质，也会提高用户接受门槛。

## 不推荐的变现方式

### 1. 广告变现

不建议把广告作为 Aura 的收入主线。

原因：

- Aura 本身覆盖在视频网站页面上，广告很容易影响观影体验。
- 浏览器扩展政策对广告展示、归属、移除能力和第三方页面干扰有较高要求。
- Aura 的品牌气质应该是低打扰陪伴，而不是新的广告层。

结论：

> 不做广告型插件。

### 2. Chrome Web Store 原生付费

不建议依赖 Chrome Web Store 原生付费能力。

原因：

- Chrome Web Store payments 已废弃。
- 后续如需收费，应使用自有支付、license、账号或兑换码体系。

结论：

> 不把 Chrome Web Store 作为支付通道，只作为分发渠道。

### 3. 用户观看数据变现

不建议出售或商业化用户观看行为数据。

原因：

- 信任风险极高。
- 浏览器扩展对浏览活动数据使用限制严格。
- Aura 更适合做本地优先、少收集数据的产品。

结论：

> 不卖数据，不做观看行为画像变现。

### 4. 未授权热门剧二创收费

不建议直接使用热门剧名、演员脸、官方视觉符号制作收费皮肤。

原因：

- 版权和肖像风险高。
- 平台审核和后续商业合作都会受影响。

可行替代：

- 做“气质映射”，例如权谋、古偶、悬疑、热血。
- 做原创角色 archetype，例如文人将军、黑猫侦探、月下贵女。

结论：

> 不做侵权二创收费，做原创角色和题材气质皮肤。

## 推荐盈利模型

## 第一层：免费核心版

免费版必须足够好用，让用户愿意长期常开 Aura。

建议免费能力：

- 默认小猫
- 黑猫侦探
- 文人将军
- 基础开关
- 调整大小
- 暂时隐藏
- 标记此刻
- 最近回看点
- 腾讯视频基础适配

免费版目标：

- 获取用户
- 建立信任
- 验证留存
- 验证皮肤切换频率

免费版不应只是残缺试用版，而应该是一个完整可用的小产品。

## 第二层：付费皮肤包

这是 Aura 最自然的第一收入来源。

可售卖方向：

- 古风权谋包
- 悬疑探案包
- 古偶甜宠包
- 热血少年包
- 雨夜黑猫高级包
- 节日限定包
- 角色主题包

建议定价：

- 单套皮肤：¥6-12
- 小主题包：¥18-28
- 早鸟终身高级皮肤：¥38-68

推荐策略：

- 先做一次性购买，不急着做订阅。
- 保持免费版好用，付费版提供更高级、更精致、更稀缺的皮肤。
- 付费皮肤不应影响插件基础功能。

为什么先卖皮肤：

- 用户容易理解。
- 与 Aura 的核心价值一致。
- 实现复杂度低于 AI、账号体系和社区。
- 不破坏低打扰观影体验。

## 第三层：定制模板 Pro

当官方皮肤体系稳定后，可以做定制模板能力。

Pro 能力可以包括：

- 上传右下角色图
- 上传左上氛围图
- 调整挂件大小
- 调整亮度和边缘融合
- 选择动效强度
- 导入皮肤包
- 导出皮肤包
- Skin Studio 本地预览

产品原则：

- 先做强约束模板，不开放完全自由画布。
- 只允许 `top-left` 和 `bottom-right` 两个运行时槽位。
- 上传前必须校验尺寸、透明背景、文件大小和构图安全区。

推荐定位：

> Aura Pro = 让用户制作自己的陪看角色。

## 第四层：创作者皮肤市场

创作者市场有长期潜力，但不适合作为早期目标。

可以在以下条件成熟后推进：

- 官方皮肤质量标准稳定
- 皮肤包格式稳定
- 上传校验和预览流程稳定
- 有一批真实用户
- 有版权审核和投诉处理能力

市场能力：

- 插画师上传皮肤包
- Aura 审核质量和版权
- 用户购买
- 创作者分成
- 平台抽成

风险：

- 审核成本
- 版权争议
- 低质量素材污染产品
- 支付和退款复杂度

结论：

> 创作者市场是中后期方向，不是 MVP。

## 第五层：官方合作与 IP 联名

这是 Aura 长期天花板最高的商业方向。

可能形式：

- 剧集宣发合作皮肤
- 视频平台活动皮肤
- 国漫 / 短剧 / 影视 IP 授权主题
- 明星或角色官方陪看形象
- 节日活动主题包

收入模式：

- 品牌活动收费
- 授权皮肤分成
- 联名皮肤售卖
- 宣发活动定制费

前提：

- Aura 已有稳定用户
- 产品体验不打扰观影
- 皮肤质量稳定
- 合规边界清楚

结论：

> 官方合作是长期增长方向，不应早于产品留存验证。

## 商业化优先级

### 阶段 1：不收费，验证留存

目标：

- 做出用户愿意常开的免费版本。
- 验证不同皮肤的使用频率。
- 验证用户是否会主动打开交互面板。
- 验证“标记此刻 / 回看点”是否有真实使用价值。

不做：

- 支付
- 账号
- 订阅
- 创作者市场

### 阶段 2：建立皮肤包机制

目标：

- 皮肤包 contract 成熟。
- Skin Studio 可预览所有皮肤。
- 内置皮肤质量稳定。
- 支持 locked / unlocked 状态，但先不接支付。

可预留字段：

```json
{
  "entitlement": "free",
  "source": "builtin",
  "locked": false
}
```

### 阶段 3：上线第一批付费皮肤

目标：

- 使用外部支付或兑换码体系。
- 小规模测试用户付费意愿。
- 先卖一次性皮肤包，不做订阅。

推荐首批付费方向：

- 高级古风人物包
- 高级悬疑黑猫包
- 节日限定包

### 阶段 4：上线定制模板 Pro

目标：

- 让用户参与宠物和角色制作。
- 提供本地导入 / 导出皮肤包。
- 开始形成用户创作资产。

### 阶段 5：探索创作者市场和官方联名

目标：

- 引入外部创作者。
- 建立审核和分成机制。
- 接触影视宣发和 IP 合作。

## 技术预留建议

短期不需要接支付，但可以提前设计 entitlement contract。

建议字段：

```json
{
  "entitlement": "free | pro | purchased",
  "source": "builtin | purchased | local",
  "locked": false,
  "license": {
    "plan": "free | pro",
    "lastCheckedAt": 0
  }
}
```

短期实现原则：

- 只做本地假数据或静态字段。
- 不接真实支付。
- 不引入账号系统。
- 不影响当前免费主链。

## 关键指标

商业化前应先观察：

- 插件安装后 7 日留存
- 用户是否长期保持 Aura 开启
- 每日皮肤切换次数
- 最常用皮肤
- 标记此刻使用次数
- 回看点点击次数
- 用户是否主动打开 Skin Studio
- 用户是否尝试导入或制作皮肤

如果这些指标不成立，不应急着收费。

## 最终结论

Aura 最好的商业化不是“工具订阅”，而是“观影氛围消费”。

它更接近：

- 浏览器插件
- 数字皮肤
- 陪看角色
- 轻量桌宠
- 影视氛围周边

推荐主线：

> 免费插件 + 高质量皮肤包 + 定制 Pro + 后期创作者市场 / 官方联名。

这条路线最符合 Aura 的产品气质、用户心理和当前技术阶段。
