# Aura 交付版路线图

> 状态：Current Delivery Plan
> 适用范围：Aura Chrome 浏览器插件主线
> 当前北极星：先把“腾讯视频陪看角色”做成一个用户愿意常开的稳定产品，再扩展模板定制、皮肤包和多站点能力。

## 0. 当前代码事实

以下内容以当前代码和测试为准，后续开发不得回退：

- Aura 当前主线是 Chrome MV3 插件，不做 Mac App / iOS App。
- 当前只官方支持腾讯视频 `v.qq.com`。
- 当前运行时是双角 PNG 皮肤：`assets.topLeft` + `assets.bottomRight`。
- 当前 `motionAssets`、`blink/react` 替换帧、`actionFrames` 已退出主线。
- 当前动效策略是单图低频动效：呼吸、轻微缩放、光感、状态降噪。
- 当前轻 utility 只保留低打扰能力：标记此刻、最近回看点、调大小、暂隐。
- 当前 Skin Studio 是皮肤验收和本地预览入口。
- 当前内置皮肤必须使用 PNG RGBA，不再使用 SVG 占位资源。

## 1. 产品定义

Aura 不是一个会抢戏的弹幕机器人，也不是一个复杂养成游戏。

Aura 当前要交付的是：
- 一个可安装的 Chrome 插件
- 一个稳定陪伴观影的小宠物
- 一个在腾讯视频播放页长期可用的低打扰体验
- 一个可继续扩到其他视频网站的运行时底座

当前一句话定义：

> Aura 是一个观影陪伴插件，在视频播放器角落放置一组高质量双角角色皮肤，通过低频动效和轻 utility 提升观影氛围而不遮挡剧情。

## 2. 路线图原则

1. 先做“可爱且稳定”，再做“聪明且复杂”
2. 先做“腾讯视频单站点打透”，再做多站点扩张
3. 先做“官方高质量皮肤”，再做模板化定制
4. 先做“轻陪伴”，不做“高打扰 AI 评论”
5. 任何新能力都不能破坏以下主链：
   - 播放页稳定识别
   - 全屏稳定
   - 快捷键稳定
   - popup 与页面状态一致
   - 关闭后无残留

## 3. 当前明确延后项

这些方向不是否定，而是明确延后：
- 宠物属性值与升级系统
- 宝可梦式复杂进化树
- LLM 对话式陪看
- 字幕 / 弹幕语义理解
- 用户公开社区和皮肤广场
- 任意图片自由上传并在线分享
- 多帧动作序列和复杂 actionFrames
- 页面字幕 / 弹幕驱动动作
- Mac App / iOS App

## 4. 交付阶段

## 阶段 0：当前基线收口

### 目标
把 Aura 当前工程从“可演示”收成“可持续迭代”的稳定底座。

### 交付物
- Chrome MV3 插件主工程可构建、可加载、可启用
- 腾讯视频运行时主链稳定
- popup、background、content runtime contract 收口
- `Skin Studio` 可用于动效和素材联调
- `smoke` 覆盖核心 contract、构建产物、scoped status

### 完成定义
- `npm run build && npm run smoke` 稳定通过
- `Command + Shift + A` / `Ctrl + Shift + A` 稳定开关
- popup 状态、页面渲染状态、storage 状态一致
- 当前运行时只保留一套主链叙事，不再混用旧黑边逻辑

### 当前判断
这一阶段已经基本成立，但后续所有阶段都默认建立在这个基线上。

## 阶段 1：腾讯视频陪看宠物 V1

### 目标
交付第一个用户能长期常开的 Aura 版本。

### 核心体验
- 打开腾讯视频播放页，宠物自动出现
- 观影过程中宠物始终低打扰、轻陪伴
- 暂停、播放、控件浮现、广告态、全屏切换都有合理反应

### 交付物
- 官方支持腾讯视频播放页
- 默认小猫稳定上线
- 悬疑 / 古偶 / 热血三套皮肤全部可切换
- 悬疑 / 古偶 / 热血三套皮肤具备清晰差异化视觉气质
- popup 完成产品化控制：
  - 总开关
  - 自动 / 手动皮肤
  - 动效强度
  - 当前状态与调试信息
- 腾讯 QA 清单可用于人工验收

### 必达验收
- 普通窗口态稳定
- 网页全屏稳定
- 原生全屏稳定
- 控件浮现时右下挂件避让
- 广告态显著弱化
- 暂停态仍有低频生命感
- 页面刷新、切集、切换 tab 后状态不串页
- `标记此刻 / 最近回看点` 只在播放器能力可用时出现
- popup 打开时能准确显示当前活动页的：
  - 剧名
  - 皮肤
  - 运行状态
  - `statusStorageKey`
  - `frameId`

### 不进入本阶段
- 复杂互动
- 文案气泡
- 用户创作入口
- 多帧动作帧系统

## 阶段 2：宠物角色感升级

### 目标
把 Aura 从“有挂件”升级成“有性格的陪看角色”。

### 核心问题
当前如果所有皮肤只是高质量静态图，用户会感到“只是贴纸”。但复杂动作帧已经被验证成本过高，因此阶段 2 只做单图 motion personality。

### 交付物
- 单图 motion personality 分层
  - 基础生命感：呼吸、轻微缩放、亮度细节
  - 状态反应：播放、暂停、控件浮现、广告态
  - 交互反应：鼠标靠近时短促增强，但不位移抢戏
- 每套皮肤拥有更明确的 motion profile
- 默认小猫补齐更自然的待机光感
- 悬疑 / 古偶 / 热血三套皮肤形成明显气质差异，但不使用多帧动作

### 阶段目标效果
- 默认：柔和陪伴
- 悬疑：冷静、观察、低频警觉
- 古偶：轻盈、优雅、细腻
- 热血：更有活力，但仍不抢戏

### 验收门槛
- 3 到 5 秒内用户能看出不同皮肤的气质差异
- 不依赖文案，单靠静态构图 + 克制动效就能体现“角色感”
- 任意状态下都不影响字幕和播放器控件使用

## 阶段 3：模板化定制 MVP

### 目标
给用户参与感，但不把 Aura 变成低质素材堆场。

### 产品策略
先做“官方模板化定制”，不直接开放完全自由上传。

### 交付物
- 宠物定制面板第一版
- 可配置项：
  - 宠物名字
  - 主题色
  - 左上氛围素材
  - 右下角色素材
  - 挂件大小
  - 低频动效强度
  - 主题包
- 官方模板素材槽位
- 本地预览能力

### 为什么这一阶段先于上传
- 质量可控
- 运行时稳定
- 用户参与感已经足够强
- 版权和审核风险最低

### 验收门槛
- 用户能在不修改运行时代码的前提下组合出自己的宠物版本
- 预览结果在 `Skin Studio` 和真实页表现一致
- 自定义结果不破坏安全区、透明边、控件避让
- 自定义结果仍遵守单图双角 contract

## 阶段 4：本地上传与皮肤包导入导出

### 目标
开放更强的用户参与，但仍保持强约束。

### 交付物
- 本地素材上传
- 受限素材包结构：
  - `top-left`
  - `bottom-right`
- 上传前校验：
  - 尺寸
  - 透明背景
  - 文件大小
  - 构图安全区
- 本地导入 / 导出皮肤包

### 明确边界
- 只支持本地自用
- 不做公共分享广场
- 不开放任意格式与任意比例
- 不开放 `blink/react/actionFrames`

### 验收门槛
- 上传失败时给出清晰原因
- 上传后在 `Skin Studio` 可直接看到结果
- 上传资产不影响主运行时性能

## 阶段 5：多站点扩展

### 目标
在不破坏腾讯主链的前提下，扩大 Aura 的可用平台。

### 站点优先级
1. 爱奇艺
2. 优酷
3. B 站长视频

### 交付物
- 每个站点一个独立 adapter
- 每个站点一份最小 QA 清单
- popup 支持展示当前站点适配状态
- runtime 保持统一 contract，不复制一套业务规则

### 扩站前提
- 腾讯主链稳定
- 多标签页状态同步稳定
- popup 对当前活动页状态读取稳定
- 皮肤运行时和站点 detector 已解耦

## 阶段 6：成长系统与轻进化

### 目标
在不打扰观影的前提下增加长期留存。

### 建议范围
- 先做“亲密度”
- 先做“陪伴成长”
- 不做复杂数值战斗属性

### 交付物
- 亲密度记录
- 成长阶段
- 成长带来的回报：
  - 新动作
  - 新表情
  - 小配饰
  - 轻进化形态

### 延后项
- 复杂属性树
- 战斗化成长
- 高负担养成任务

## 5. 三条并行工作流

为了保证交付节奏，后续开发默认沿三条工作流推进。

### A. 运行时主链
- 站点识别
- 播放器容器定位
- 全屏与控件避让
- popup 与状态链一致性

### B. 角色与动效
- 默认宠物低频动效打磨
- 皮肤差异化 motion preset
- 广告态 / 亮场 / 暂停态表现优化

### C. 模板与创作系统
- 素材规范
- 模板槽位
- 皮肤预览台
- 定制与上传系统

## 7. 每个阶段的交付门禁

每个阶段在进入下一阶段前，都必须同时满足以下门禁：

1. `npm run build && npm run smoke` 通过
2. `npm run qa:extension` 和 `npm run qa:reload` 在相关 runtime 改动后通过
3. popup、页面、storage 三者状态一致
4. 快捷键开关稳定，无残留
5. 新能力不破坏现有皮肤 contract
6. 不并行运行会同时重建 `dist/` 的命令

## 8. 近期执行顺序

当前建议的实际推进顺序如下：

1. 固化当前稳定基线并提交。
2. 完成腾讯视频陪看角色 V1 的最终实机稳定。
3. 打磨默认小猫，让它成为真正可长期常开的主角色。
4. 做完官方皮肤的单图 motion personality 分化。
5. 上线模板化定制 MVP。
6. 再进入本地上传和导入导出。
7. 最后扩多站点和成长系统。

## 9. 当前产品结论

Aura 当前最应该坚持的方向不是“更像 AI”，而是“更像陪你看剧的氛围角色”。

所以后续路线图排序固定为：
- 稳定陪伴
- 角色感
- 模板定制
- 本地创作
- 多站点扩张
- 长线成长

只要这条顺序不乱，Aura 就会越来越像一个完整产品，而不是不断加功能的插件实验品。
